Zero risk to state rules. Every selector component (ModelSelector, ToolsSelector, SubAgentSelector, MCPSelector, SystemPromptEditor) is a self-contained React component. The enable/disable logic lives inside each component via props + Zustand stores. Moving them from <Header> to <SettingsPanel> is just changing the parent div — props and state flow unchanged. Implementation can be incremental: move one control at a time, verify rules after each step.
Implementation order: (1) Create empty SettingsPanel shell + gear toggle, (2) Move model selector → verify, (3) Move tools/agents → verify mutex, (4) Move remaining controls, (5) Add stop button to tab bar, (6) Move mode to chips row, (7) Move pins button to tab bar, (8) Delete Header component.
